# ADM-401: Admin Route Guard

**Epic:** ADM-400 Admin Application  
**Status:** 🔘 Todo  
**Priority:** P0 — Blocks all authenticated admin flows

---

## Story

> As a security-conscious developer, I want all admin dashboard routes to be protected by an auth guard so that unauthenticated users are automatically redirected to the login page if they attempt to access a guarded URL directly.

---

## Acceptance Criteria

- [ ] An `AuthGuard` (`auth.guard.ts`) is created using Angular's functional guard pattern (`CanActivateFn`).
- [ ] The guard reads auth state from `CoreService.authStatus()` (Signal).
- [ ] If `authStatus === 'unauthenticated'`, the guard redirects to `/authentication/login`.
- [ ] If `authStatus === 'loading'`, the guard waits (returns an observable/promise) until the state resolves.
- [ ] The `/dashboard` route group in `app.routes.ts` is wrapped with `canActivate: [AuthGuard]`.
- [ ] Authentication and public routes (`/splash`, `/authentication/**`) remain unguarded.
- [ ] Direct navigation to `/dashboard/home` while logged out redirects to `/authentication/login`.
- [ ] `data-test-id="auth-guard-redirect"` is verifiable in E2E (guard fires → login page shown).

---

## Technical Notes

```typescript
// auth.guard.ts (functional guard pattern)
export const AuthGuard: CanActivateFn = () => {
  const core = inject(CoreService);
  const router = inject(Router);
  // Use toObservable to wait for 'loading' to resolve
};
```

- Use `toObservable(core.currentUser)` + `filter(u => u !== undefined)` to avoid guarding before Firebase resolves.
- Do **not** use class-based guards (deprecated in Angular 15+).
