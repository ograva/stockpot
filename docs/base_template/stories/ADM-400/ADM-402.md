# ADM-402: User Management List

**Epic:** ADM-400 Admin Application  
**Status:** ✅ Done  
**Priority:** P1

---

## Story

> As an admin, I want to see a paginated list of all registered users from the Firestore `users` collection so that I can monitor who has access to the system and act on individual accounts.

---

## Acceptance Criteria

- [x] A `/dashboard/users` route is added to `pages.routes.ts`.
- [x] `UsersComponent` (`pages/users/users.component.ts`) renders a Material `mat-table`.
- [x] Table columns: **Name**, **Email**, **Role**, **Created Date**, **Actions**.
- [x] Data is loaded from Firestore `users` collection via a `UsersService`.
- [x] Clicking **Edit** on a row navigates to `/dashboard/users/:id/edit` (ADM-404).
- [x] A **"+ New User"** button navigates to `/dashboard/users/new` (ADM-404).
- [x] Table supports client-side **search/filter** by name or email.
- [x] An empty state message is shown when no users exist.
- [x] Navigation item **"Users"** is added to `sidebar-data.ts` under an `Admin` nav cap.
- [x] All interactive elements have `data-test-id` attributes (e.g., `users-table`, `user-edit-button-{{id}}`, `users-new-button`).

---

## Firestore Schema

Collection: `users`

```
users/{uid}
  ├── uid: string
  ├── name: string
  ├── email: string
  ├── role: 'admin' | 'user'
  └── createdAt: Timestamp
```

---

## Technical Notes

- Use `collectionData(collection(firestore, 'users'), { idField: 'uid' })` from `@angular/fire/firestore`.
- The `UsersService` should be `providedIn: 'root'` and return a Signal via `toSignal()`.
- Role column should render a `mat-chip` with colour coded by role (`admin` = primary, `user` = default).
