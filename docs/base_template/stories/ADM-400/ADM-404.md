# ADM-404: User Create / Edit Form

**Epic:** ADM-400 Admin Application  
**Status:** ✅ Done  
**Priority:** P1

---

## Story

> As an admin, I want to create new users and edit existing user profiles in a dedicated form page so that I can manage account details and roles without navigating away from the admin app.

---

## Acceptance Criteria

### Create Mode (`/dashboard/users/new`)

- [x] Form is pre-populated with empty fields.
- [x] **Name**, **Email**, **Password**, and **Role** fields are required.
- [x] Submitting calls `createUserWithEmailAndPassword` via Firebase Auth, then writes a `users/{uid}` document to Firestore.
- [x] On success, navigates back to `/dashboard/users` with a success snackbar.

### Edit Mode (`/dashboard/users/:id/edit`)

- [x] Form is pre-populated from the Firestore `users/{id}` document.
- [x] **Email** is read-only in edit mode (Firebase Auth email changes are a separate flow).
- [x] **Password** field is hidden in edit mode.
- [x] Submitting updates the Firestore `users/{id}` document.
- [x] A **"Delete User"** button (with confirmation dialog) removes the Firestore document.
- [x] On success, navigates back to `/dashboard/users` with a success snackbar.

### Both Modes

- [x] Inline validation errors on dirty + invalid fields.
- [x] **"Cancel"** button navigates back to `/dashboard/users` without saving.
- [x] All inputs have `data-test-id` attributes (e.g., `user-form-name-input`, `user-form-email-input`, `user-form-role-select`, `user-form-save-button`, `user-form-cancel-button`, `user-form-delete-button`).

---

## Firestore Schema

Writes to `users/{uid}`:

```
{
  uid: string,
  name: string,
  email: string,
  role: 'admin' | 'user',
  createdAt: Timestamp   // set on create only
}
```

---

## Technical Notes

- Detect mode via `ActivatedRoute.snapshot.paramMap.get('id')` — if `'new'` then create mode.
- Reuse `UsersService` from ADM-402 for reads; add `createUser()` and `updateUser()` methods.
- Use `MatDialog` with a confirmation component for delete action.
