# SYS-104: Centralise Firebase Auth Listener in AppComponent

**Epic:** SYS-100 System Foundation
**Status:** ✅ Complete
**Priority:** P1 — Improves reliability of session transitions across all pages

---

## Story

> As a developer, I want the Firebase Auth listener registered once in `AppComponent`
> so that login, logout, and session-expiry events are handled consistently at the
> top level without any component having to subscribe to Firebase independently.

---

## Context

Currently, `CoreService` owns the Firebase Auth subscription via `toSignal(authState(...))`.
While functional for initial page load, this scatters auth-driven routing logic across
`SplashComponent`, `AuthGuard`, and indirectly the header — leaving session *transitions*
(a user logging out mid-session, or a token expiry) unhandled at the app level.

The correct pattern is a single `onAuthStateChanged` listener in `AppComponent` that:
1. Writes auth state into a shared `WritableSignal` on `CoreService`
2. Drives top-level navigation on every subsequent auth state change

---

## Acceptance Criteria

- [x] `AppComponent` registers `onAuthStateChanged()` in its constructor and unsubscribes `ngOnDestroy`
- [x] `CoreService` no longer imports `Auth` or calls `authState()`; `currentUser` is a `WritableSignal` written by `AppComponent`
- [x] `CoreService.authStatus` is a `computed()` derived from `currentUser` (not a method)
- [x] On **login** event (post first-emission): `AppComponent` navigates to `/dashboard/home` if currently on `/authentication/*` or `/splash`
- [x] On **logout / token expiry** event: `AppComponent` navigates to `/authentication/login` if currently on `/dashboard/*`
- [x] `SplashComponent` still handles the initial routing (2s timer + authStatus check); it no longer needs to manage auth subscriptions itself
- [x] `AuthGuard` continues to function unchanged — it reads `CoreService.currentUser` signal
- [x] TypeScript compiles clean (zero errors)

---

## Technical Notes

- `isFirstEmission` flag in `AppComponent` ensures the Firebase boot emission does not fight with `SplashComponent`'s initial navigation
- `CoreService.currentUser` must be exposed as a **readonly** signal externally (`WritableSignal.asReadonly()`) to prevent accidental writes from other consumers
- `CoreService.setCurrentUser(user: User | null)` is the only mutation path, called exclusively by `AppComponent`
