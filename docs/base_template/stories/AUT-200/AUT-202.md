# AUT-202: Global Auth UI Toggles

**Epic:** AUT-200 Authentication & Initialization  
**Status:** ✅ Done  
**Priority:** P1

---

## Story

> As a user, I want the header and sidebar to dynamically reflect my authentication state so that I can clearly see whether I am logged in and quickly access login or logout actions from anywhere in the app.

---

## Acceptance Criteria

### Header

- [x] A **user avatar / icon** in the top-right corner of the header is always visible.
- [x] When **unauthenticated**: icon shows a generic person icon; clicking navigates to `/authentication/login`.
- [x] When **authenticated**: icon shows the user's display name initial or `photoURL` avatar; clicking opens a dropdown menu.
- [x] Dropdown menu (authenticated) contains: **"My Profile"** → `/dashboard/profile` and **"Logout"** action.
- [x] `data-test-id="header-user-menu-button"`, `data-test-id="header-logout-button"`, `data-test-id="header-login-button"`.

### Sidebar

- [x] A **"Login"** nav item is shown when unauthenticated.
- [x] A **"Logout"** nav item is shown when authenticated (replaces Login item).
- [x] Clicking **Logout** calls `signOut()` from Firebase Auth and redirects to `/authentication/login`.
- [x] `data-test-id="sidebar-login-link"`, `data-test-id="sidebar-logout-button"`.

### Signal Reactivity

- [x] All toggles use `CoreService.authStatus()` Signal — no manual subscriptions.
- [x] Changes take effect immediately without requiring a page reload.

---

## Technical Notes

- Use `@if (authStatus() === 'authenticated')` in templates (Angular control flow).
- Logout action: `inject(Auth).signOut()` then `router.navigate(['/authentication/login'])`.
- Keep auth logic in the header/sidebar components; do not create a separate auth service wrapper.
