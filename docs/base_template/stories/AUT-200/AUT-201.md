# AUT-201: Launch Splash Screen

**Epic:** AUT-200 Authentication & Initialization  
**Status:** ✅ Done  
**Priority:** P1

## Story Description
As a user, I want to see a branded splash screen upon opening the app so that I know it is initializing and I am not left with a blank white page.

## Acceptance Criteria
- [x] Shows the Novus Apps logo centered on the screen.
- [x] Checks the current Firebase Auth status (`onAuthStateChanged` via `CoreService.authStatus` signal).
- [x] Redirects to `/dashboard/home` (if authenticated) or `/authentication/login` (if unauthenticated) after at least 2 seconds.
- [x] Animation/Transition into the landing page is smooth.

### Implementation Notes
- `SplashComponent` uses a `minDisplayElapsed` signal set by a 2-second `setTimeout`.
- An `effect()` watches both `minDisplayElapsed` and `core.authStatus()` and only navigates when both conditions are met.
- Integrated as the root route in `app.routes.ts`.
