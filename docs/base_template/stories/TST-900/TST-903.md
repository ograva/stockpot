# TST-903: Admin Registration E2E

## Story Description

As a QA engineer, I want automated E2E tests covering the Admin registration form so that form validation, duplicate-email handling, happy-path registration, and navigation links are all regression-tested.

## Acceptance Criteria

- [ ] Registration page at `/authentication/register` renders all form elements (`register-name-input`, `register-email-input`, `register-password-input`, `register-submit-button`).
- [ ] Submitting an empty form triggers Angular reactive form validation on at least one field.
- [ ] Submitting with an already-registered email displays `[data-test-id="register-error-message"]`.
- [ ] Submitting with a unique email and valid password creates the user and redirects to `/dashboard`.
- [ ] Clicking the `[data-test-id="register-login-link"]` navigates to `/authentication/login`.
- [ ] The happy-path test uses a `Date.now()`-unique email so it is idempotent across repeated runs.

## Definition of Done

- [ ] `T003-register.spec.ts` exists at `e2e/admin/flows/authentication/`.
- [ ] All 5 sub-tests (T003.1–T003.5) pass against the Firebase Auth emulator.
- [ ] Duplicate-email test reuses `TEST_USER` from `e2e/global-setup.ts` — no hardcoded strings.
- [ ] `data-test-id="register-form"` is present on the `<form>` element in `side-register.component.html`.
