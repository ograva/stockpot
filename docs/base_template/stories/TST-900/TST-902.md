# TST-902: Admin Logout E2E

## Story Description

As a QA engineer, I want automated E2E tests covering the Admin logout flow so that any regression in Firebase `signOut`, sidebar auth-state toggling, or post-logout navigation is caught immediately.

## Acceptance Criteria

- [ ] Sidebar shows `Logout` item and hides `Login` item when a user is authenticated.
- [ ] Clicking the sidebar `Logout` item calls Firebase `signOut` and redirects to `/authentication/login`.
- [ ] After logout, the sidebar resets to guest state: `Login` visible, `Logout` hidden.
- [ ] Tests clean up cookies and localStorage after each run (no cross-test pollution).
- [ ] All assertions use `data-test-id` selectors (`sidebar-nav-logout`, `sidebar-nav-login`, `login-form`).

## Definition of Done

- [ ] `T002-logout.spec.ts` exists at `e2e/admin/flows/authentication/`.
- [ ] All 3 sub-tests (T002.1, T002.2, T002.3) pass against the Firebase Auth emulator.
- [ ] Tests depend on `loginAsTestUser()` from `e2e/helpers/auth.helper.ts` — no duplicated login logic.
