# TST-904: Admin Splash Screen Transition E2E

## Story Description

As a QA engineer, I want automated E2E tests verifying the Admin splash screen renders correctly and routes to the right destination based on Firebase Auth state, so that the core initialization flow is regression-tested.

## Acceptance Criteria

- [ ] Navigating to `/splash` renders `[data-test-id="splash-screen"]`, `[data-test-id="splash-logo"]`, and `[data-test-id="splash-loader"]`.
- [ ] An unauthenticated user navigating to `/` is redirected to `/authentication/login` within 10 seconds.
- [ ] An authenticated user navigating to `/splash` is redirected to `/dashboard/home` within 10 seconds.
- [ ] After the authenticated redirect, `[data-test-id="header-app-title"]` is visible (confirms full layout loaded).

## Definition of Done

- [ ] `T010-splash.spec.ts` exists at `e2e/admin/flows/splash/`.
- [ ] All 3 sub-tests (T010.1, T010.2, T010.3) pass against the Firebase Auth emulator.
- [ ] T010.3 uses `loginAsTestUser()` from `e2e/helpers/auth.helper.ts` to establish session before splash navigation.
