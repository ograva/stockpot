# USR-503: First-Login Onboarding Flow

**Epic:** USR-500 User Application  
**App Scope:** `projects/user-app` only  
**Status:** ✅ Done  
**Priority:** P2 — Enhances USR-502

---

## Story

> As a newly registered user, I want to be guided through a short profile setup step after my first login so that I can complete my account details before reaching the home page.

---

## Acceptance Criteria

- [x] After a successful registration or first login, if the user's Firestore `users/{uid}` document does **not** have a `name` set, the app redirects to `/dashboard/profile?onboarding=true` (logic in `SplashComponent`).
- [x] In onboarding mode, the profile page shows a **"Complete your profile"** heading.
- [x] **Display Name** is required in onboarding mode (cannot be skipped).
- [x] A **"Get Started"** submit button (instead of "Save") completes the step and navigates to `/dashboard/home`.
- [x] Returning users (who already have a `name`) skip this step entirely on login.
- [x] `data-test-id="onboarding-name-input"`, `data-test-id="onboarding-submit-button"` are present for E2E targeting.

---

## Technical Notes

- Detection logic: in `CoreService` or the splash effect, after auth resolves, check `users/{uid}.name` — if absent, redirect to `/dashboard/profile?onboarding=true`.
- The `ProfileComponent` reads `ActivatedRoute.queryParamMap` for `onboarding=true` to toggle the mode.
- No separate component needed — reuse `ProfileComponent` (USR-502) with mode flag.
