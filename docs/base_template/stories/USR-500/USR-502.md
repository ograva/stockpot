# USR-502: User Profile Form

**Epic:** USR-500 User Application  
**App Scope:** `projects/user-app` only — this is a self-service form for end-users. Admin user management is covered by ADM-402/ADM-404.  
**Status:** ✅ Done  
**Priority:** P1

---

## Story

> As an end-user, I want a profile page where I can view and update my personal information so that the system has accurate details about me and I can manage my own account.

---

## Acceptance Criteria

- [x] A `/dashboard/profile` route is added to `pages.routes.ts`.
- [x] `ProfileComponent` (`pages/profile/profile.component.ts`) loads the current user's data from `CoreService.currentUser()` and the `users/{uid}` Firestore document.
- [x] The form contains: **Display Name**, **Email** (read-only), **Photo URL** (optional text input), **Role** (read-only badge).
- [x] Saving updates the `users/{uid}` Firestore document AND calls `updateProfile()` on the Firebase Auth user.
- [x] Changes are persisted offline via `StoreForwardService` Store-and-Forward (DAT-301).
- [x] A success snackbar confirms save.
- [x] Navigation item **"My Profile"** is added to `sidebar-data.ts` (hidden for unauthenticated users).
- [x] All fields carry `data-test-id` attributes (`profile-name-input`, `profile-save-button`, `profile-form`).
- [x] The **header** profile icon/avatar reflects the updated display name or photo once saved (reactive `avatarUrl` and `displayName` computed signals in `HeaderComponent`).

---

## Firestore Schema

Reads/writes to `users/{uid}`:

```
{
  uid: string,
  name: string,
  email: string,       // read-only
  photoURL?: string,
  role: string         // read-only
}
```
