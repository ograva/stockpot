# DAT-301: DataShareService — Two-Stage Store-and-Forward

**Epic:** DAT-300 Data Management  
**Status:** ✅ Done  
**Priority:** P1 — Blocks USR-502 (Profile Form) and all offline-capable data flows

---

## Story

> As a developer, I want a `DataShareService` that implements the two-stage Store-and-Forward pattern so that all apps can read and write data optimistically to LocalStorage first, then sync to Firestore in the background, enabling an offline-first experience.

---

## Acceptance Criteria

- [x] Service implemented as `StoreForwardService` at `projects/user-app/src/app/services/store-forward.service.ts` (singleton, `providedIn: 'root'`).
- [x] Writing data:
  1. Updates a `WritableSignal` immediately (optimistic UI update).
  2. Writes to `localStorage` synchronously inside `set()`.
  3. Queues a Firestore `setDoc` in the background (fire-and-forget with console error logging).
- [x] Reading data:
  1. `get(key)` hydrates from `localStorage` on first access (fast first paint).
  2. `sync(key, firestorePath)` subscribes to Firestore via `onSnapshot` for the canonical value; signal is updated when remote differs.
- [x] The service is `providedIn: 'root'` (singleton).
- [x] Exported methods:
  - `get<T>(key): WritableSignal<T | null>`
  - `set<T>(key, value, firestorePath?): void`
  - `sync(key, firestorePath): void`
  - `unsync(key): void`
  - `clear(key): void`
- [ ] Unit tests cover: optimistic write, LocalStorage hydration, Firestore sync, and conflict handling.

> **Note:** The service is named `StoreForwardService` (not `DataShareService`). A separate `DataShareService` exists for secure in-memory parameter passing between components (not related to this pattern).

## Technical Notes

```typescript
// Conceptual pattern
@Injectable({ providedIn: "root" })
export class DataShareService {
  private store = new Map<string, WritableSignal<unknown>>();

  set<T>(key: string, value: T, firestorePath?: string) {
    // 1. Signal update
    this.signal(key).set(value);
    // 2. LocalStorage
    localStorage.setItem(key, JSON.stringify(value));
    // 3. Background Firestore write (if path provided)
    if (firestorePath) {
      /* setDoc(...) */
    }
  }
}
```

- Use `nanoid` (or `short-uuid`) — already in `allowedCommonJsDependencies` or add it.
- Firestore offline persistence (`enableIndexedDbPersistence`) handles network-level caching; DataShareService handles application-level optimistic state.
